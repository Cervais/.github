# Security Policy

Cervais builds cybersecurity products for critical infrastructure, so we take vulnerability reports seriously and appreciate the work of security researchers.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, report them privately:

- Email: security@cervais.com
- If the issue affects a specific repository, you can also use that repository's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) feature, if enabled.

When reporting, please include as much of the following as you can:

- A description of the vulnerability and its potential impact
- Steps to reproduce, including affected product, service, or repository
- Any proof-of-concept code, logs, or screenshots
- Your assessment of severity

## What to Expect

- We will acknowledge your report within **3 business days**.
- We will investigate and provide an initial assessment within **10 business days**.
- We will keep you informed as we work on a fix and coordinate disclosure timing with you.
- We ask that you give us a reasonable opportunity to address the issue before any public disclosure.

## Scope

This policy covers Cervais-owned repositories under the [cervais](https://github.com/cervais) GitHub organization, as well as Cylenium Cloud and the Guardian Edge (SG‑X) product line. For vulnerabilities in third-party dependencies, please also report them upstream where applicable.
