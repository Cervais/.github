# Contributing

Thanks for your interest in contributing to a Cervais repository.

## Before you start

- For anything beyond a small fix, please open an issue first to discuss what you'd like to change.
- Security vulnerabilities should **not** be reported as issues — see [`SECURITY.md`](SECURITY.md) instead.
- By contributing, you agree to follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Making a change

1. Fork the repository and create a branch from `main`.
2. Make your change, keeping the diff focused on a single concern.
3. Add or update tests where applicable.
4. Make sure existing tests and linters pass.
5. Open a pull request using the provided template, describing what changed and why.

## Pull requests

- Keep pull requests small and scoped; large, unrelated changes are harder to review and merge.
- Link any related issue in the description.
- A maintainer will review your PR and may request changes before merging.

## Questions

If something isn't covered here, open an issue in the relevant repository or reach out at contact@cervais.com.
